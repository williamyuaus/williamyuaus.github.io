Disable the Error Notification Toast When Generating PDF From a Vue Project

William Yu included in Vue Debug
 2023-08-11  359 words   2 minutes 
Contents
  • Intro
  • What I tried
  • Solution

Intro

If we need to generate a PDF from a webpage by puppeteer, we do not want to display any error notification popup on it. That’s the situation I was working on. It is a Vue Project. On the browser, when a REST API call failed, there would be showing a notification popup on the screen. If we need to generate a PDF version based on the webpage, the popup will stick on the output. There is an example of it:

What I tried

The solution of my approach is to use media query. Initially, I chose the screen size the condition to disable the notification. I added the CSS to App.vue:

<style>
@media only screen and (max-width: 596px) {
  .toastNotification {
    display: none !important;
  }
}
</style>

The max-width: 596px means that the size of the PDF report is 595px * 842px. In this project, we use vue-toastification for the toast library. In the main.ts, we defined this:

...
import Toast from 'vue-toastification';
...
app.use(Toast, {
  transition: 'Vue-Toastification__fade',
  maxToasts: 20,
  newestOnTop: true,
  toastClassName: 'toastNotification',
});

However, it didn’t work. I though I need some extra configuration for Puppeteer. I tried setViewPort and width and length forpage.pdf function.

What I tried included:

const browser = await puppeteer.launch({
        executablePath: '/usr/bin/google-chrome',
        headless: 'new', args: ['--no-sandbox', '--disable-setuid-sandbox'],
        defaultViewport: null
    });
const page = await browser.newPage();

await page.setViewport({ width: 595, height: 842 });
await page.goto(url, {waitUntil: 'networkidle0', timeout: 0});

...


const pdfBuffer = await page.pdf({
    printBackground: true,
    format: 'A4',
    width: '595px',
    height: '842px',
    displayHeaderFooter: true,
    margin: { right: '20px', left: '20px', top: '130px', bottom: '80px' },
    headerTemplate: header,
    footerTemplate: footer
});

Unfortunately, all of these didn’t work. The generated PDF always got the error notification on it.

Solution

Finally, I realised that the process of page.pdf in Puppeteer emulates a “print to PDF” action. So, no matter what screen size the puppeteer goes to, it would be always ignored by page.pdf process.

For the solution, we change the media query in App.vue to:

<style>
@media print {
  .toastNotification {
    display: none !important;
  }
}
</style>

Once this change is done, there would be no more error notification toast showing on the generated PDF.
